== 本次完成(LiveKit backend 端到端,已验证) ==
- 用本地 `livekit-server --dev`(brew install livekit;凭据 devkey/secret)做端到端验证,
  并且**也对真 LiveKit Cloud**(wss://lerobot-zt6uc2pv.livekit.cloud)验证通过。
  (沙箱里 `uv` 连 Cloud 报 UnknownIssuer,但 livekit Rust FFI 客户端连 Cloud 正常。)
- 修掉两个真 bug(之前 controller 端 get_observation() TimeoutError 的根因):
  1. `room.connect()` **不触发 "connected" 事件** → connect() 返回后直接 `self.connected.set()`。
  2. seq↔帧 **FIFO 1:1 配对是错的**:视频流有丢帧(实测 ~46% 到达),FIFO 会越漂越旧
     (末帧标成 ~80 帧前的 seq,约 2.7s 陈旧,且不会超时只会静默配错)。
     → 改成订阅端用 `_SEQ_TOPIC` 上**最新** seq 给到达帧打标(monotonic + 新鲜,实测帧 seq 与
     state seq 只差 1 帧)。auto_subscribe 默认 True,track_subscribed 正常触发(2a 排除)。
- close() 取消 _consume task,消除断开时 "Task was destroyed but it is pending"。
- mac_daemon:`--signaling-url` 改成非必填(livekit 自己做信令);加 per-backend 必填校验
  (aiortc 需 signaling-url;livekit 需 url+token)。run_daemon 对 livekit 传 signaling=None;
  CaptureAgent 容忍 signaling=None。
- cloud_teleop_so100.py 加 `--transport/--signaling-url/--livekit-url/--livekit-token`。
- 新增 opt-in 集成测试 tests/robots/test_webrtc_proxy_livekit.py(无 LEROBOT_LIVEKIT_URL 时 skip;
  daemon 走子进程,本进程当 controller,断言 obs/action 往返 + obs 在推进)。
- 文档:transport_livekit.py docstring 去掉 "UNTESTED SCAFFOLD"/`# VERIFY`,写清 _SEQ_TOPIC 方案;
  DESIGN §11.1 标注 LiveKit 已验证 + 新增 §11.1.1 NAT/受限出网(http_proxy)可达性矩阵
  (为什么选 SFU 而非 aiortc P2P);README 加「LiveKit backend(experimental)」运行+测试段落。
